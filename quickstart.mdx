Get your backend up and running in minutes with InsForge. Choose between cloud-hosted or self-hosted deployment.

## Option 1: Cloud-hosted (Recommended)

The fastest way to get started. No setup required.

<Steps>
  <Step title="Create an account">
    Visit [insforge.dev](https://insforge.dev) and sign up for a free account.
  </Step>

  <Step title="Create a project">
    Once logged in, create your first project. You'll get:
    - A PostgreSQL database
    - Authentication with OAuth providers
    - S3-compatible storage
    - Serverless functions runtime
    - AI integration
  </Step>

  <Step title="Get your credentials">
    From your project dashboard, copy:
    - **Base URL**: `https://your-app.region.insforge.app`
    - **Anon Key**: Your public API key
  </Step>

  <Step title="Connect InsForge MCP">
    Follow the [MCP Setup Guide](/mcp-setup) to connect your AI assistant to InsForge.
  </Step>
</Steps>

## Option 2: Self-hosted with Docker

Run InsForge on your own infrastructure.

<Steps>
  <Step title="Prerequisites">
    Install [Docker](https://www.docker.com/) and [Docker Compose](https://docs.docker.com/compose/install/)
  </Step>

  <Step title="Clone and configure">
    <CodeGroup>
    ```bash Clone repository
    git clone https://github.com/insforge/insforge.git
    cd insforge
    ```

    ```bash Configure environment
    cp .env.example .env
    # Edit .env and set:
    # - JWT_SECRET (generate with: openssl rand -hex 32)
    # - POSTGRES_PASSWORD
    # - Other required variables
    ```
    </CodeGroup>
  </Step>

  <Step title="Start services">
    ```bash
    docker compose -f docker-compose.prod.yml up -d
    ```
    
    Wait for all containers to be healthy (1-2 minutes):
    ```bash
    docker compose ps
    ```
  </Step>

  <Step title="Access the dashboard">
    Open [http://localhost:7130](http://localhost:7130) in your browser.
    
    Your API base URL will be: `http://localhost:7131`
  </Step>

  <Step title="Connect InsForge MCP">
    Follow the [MCP Setup Guide](/mcp-setup) using the self-hosted configuration.
  </Step>
</Steps>

<Note>
  For production deployment, SSL/TLS, domain configuration, and detailed environment variables, see the [Self-Hosting Guide](/platform/self-hosting).
</Note>

## Build your first app with AI

Once your backend is running and MCP is connected, use AI to build your first application:

<Steps>
  <Step title="Prompt your AI assistant">
    ```
    I want to build a todo app with InsForge. Create:
    1. A todos table with title, completed, and user_id columns
    2. Row Level Security so users only see their own todos
    3. A simple React frontend with authentication
    ```
  </Step>

  <Step title="Let AI do the work">
    Your AI assistant will:
    - Use MCP to create the database table
    - Configure Row Level Security policies
    - Generate a React app with the InsForge SDK
    - Set up authentication flows
  </Step>

  <Step title="Run and test">
    The AI will provide instructions to run your app locally and test it.
  </Step>
</Steps>
